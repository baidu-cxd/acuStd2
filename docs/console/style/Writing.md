# 文案

界面中说明和提示文案能有效的指导用户的操作及带给用户更好的可读性，文案要符合整个产品的气质和目标用户，建立产品的专业感，加强用户对我们的信任。

[[toc]]

## 语言

语言是是组成构成界面的重要部分，也是和用户沟通的基础。在界面中应考虑如何简洁明了的传达，让用户立刻能懂，欣然接受。

**1. 精简准确的表达**

压缩文案，在涵义不变的情况下，优先选择最简洁、字数最少的内容；去掉与用户无关、对用户无用的文字；完整而准确，每一个字都要有意义，让用户能够快速获取信息。

![图片](http://baiduyun-guideline.bj.bcebos.com/console/style/Writing/01_2x.png)

**2. 贴近用户的语言**

使用用户熟悉的语言，采用主流词汇和术语，尽可能让绝大部分用户都能理解，不要使用太生僻或过分专业的用词。


![图片](http://baiduyun-guideline.bj.bcebos.com/console/style/Writing/02_2x.png)

**3. 语言风格一致**

区别于用户产品的轻松活跃感，百度云作为以技术为支撑的产品，想传递的是一种简洁、严谨、专业的风格。

为了避免降低专业性的印象，在表达过程中不要使用太口语化的语言，包括俗语和网络语言。

![图片](http://baiduyun-guideline.bj.bcebos.com/console/style/Writing/03_2x.png)

**4. 表述具有统一性**

描述同一个事物要用词一致；操作名称和目标页面标题名称需一致。


![图片](http://baiduyun-guideline.bj.bcebos.com/console/style/Writing/04_2x.png)

**5. “先说目的”法则**

从用户角度出发，先说明操作的目的和重要性，能促使用户更愿意去执行。

![图片](http://baiduyun-guideline.bj.bcebos.com/console/style/Writing/05_2x.png)

**6. “问题-后果-方案”法则**

当用户在方案A和B之间做一个选择，而我们希望用户选择方案B（因为A有很严重的后果C）的时候。一般是写提出问题，即如果采用方案A会有后果C，然后建议采用方案B，最后把问题抛给用户自己选择。

比如：当方案A-“退出编辑”，后果C-“无法保存所输入的信息”，方案B-“先去保存”。


![图片](http://baiduyun-guideline.bj.bcebos.com/console/style/Writing/06_2x.png)

## 语气

语言主要带来的是表达的内容，语气主导了与用户表达过程中的情感和氛围，根据产品类型决定产品是活跃还是严谨专业的表达。

**1. 友好、乐观、正面、尊重用户**

尽量不要出现 不、没有、未、必须等字眼。如，未绑定、过期等。

![图片](http://baiduyun-guideline.bj.bcebos.com/console/style/Writing/07_2x.png)

**2. 使用第二人称拉近用户距离 ，直接对话**

使用第二人称[你]，能够体现与用户的近距离感，更有利于我们指引和鼓励用户操作；第一人称主要用在用户专属的区域，如“我的工单” 。

![图片](http://baiduyun-guideline.bj.bcebos.com/console/style/Writing/08_2x.png)

**3. 谨慎使用感叹号[!]和波纹线[~]， 表述尽量平和**

[!]感叹号会让文案显的非常激动和情感化，营造紧张气氛，不利于和用户友好的互动和专业化的表达；[~]波纹线会让文案显得些许调皮，与百度云的严谨风格不搭，建议避免使用。

![图片](http://baiduyun-guideline.bj.bcebos.com/console/style/Writing/09_2x.png)
