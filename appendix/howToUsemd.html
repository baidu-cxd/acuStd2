# 头文件规定

在 Markdom 文档的开始，请用以下语法规定文档的名称，创建者的信息是网站上不可见的，主要用于便于管理：

---
title: 页面的标题
author: 创建者
sidebarDepth: 0
---

例如：本篇文档的名称：

---
title: Md 文档编写说明（初级）
author: Yihan
sidebarDepth: 0
---

sidebarDepth: 0 是用于规定侧边栏导航样式的代码，详细见文档 MD 文档编写说明（高级）

# 页面内导航

[[toc]]

在合适的位置使用 [[toc]] 会创建一个页内导航，自动抓取 H2 和 H3 标签的内容构造导航结构

注意

我们以前在 H2 和 H3 上手动添加锚点，然后用无序列表构造导航，这种方式比较麻烦，而且会污染无序列表的样式，因此该方法被废除

# 标题

H1 标题的语法如下：

# 我是页面标题，注意 “#” 和文字间有一个空格

注意

H1 一个页面只能用一次，表示整个页面的名称

H2/H3 标题的语法以此类推：

## 我是 H2 标题，注意 “##” 和文字间有一个空格
### 我是 H3 标题，注意 “###” 和文字间有一个空格

# [新] 色块生成

这个组件用于规范色彩的展示

::: color #108cee
这是主色
:::

#108cee

这是主色

如果颜色太浅，白色文字无法阅读：

::: color-l #eeeeee
这是主色
:::

#eeeeee

这是浅色

如果想要制作一个色阶的方法，使用 <div class="wrp" id="颜色"></div>将色块包容：

<div class="wrp" id="blue">

::: color #1c1c1c
这是黑色
:::

::: color #333333
这是灰色
:::

::: color-l #e6e6e6
浅灰色背景下有蓝色文字
:::

</div>

#1c1c1c

这是黑色

#333333

这是灰色

#e6e6e6

这是浅灰色

控制浅色块下文字颜色的方式是在 .md 文档的底部添加一个样式代码：

<style>
div.wrp#blue .colorBox.light p{
color:#108cee;
}
</style>

注意

div.wrp# 后的内容（也就是上面例子中的 blue ）需要和 <div class="wrp" id="blue"> 中的 id 保持一致
